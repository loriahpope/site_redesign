Idea Submission Site

The Idea Portal currently exists as a platform for employees to submit innovative solutions that will improve the operations of the company. A similar idea submission platform exists on the company's internal social networking site, where employees identify potential areas within the company that can be improved. Ideally, the crowdsourcing, social aspect of the internal social networking solution can be combined with the structure and process definition within the existing Idea Portal to create an ideal user experience.

I was the lead designer on this project. In addition to completing a heuristic review of the existing portal, I interviewed many users of the current site to understand their pain points and difficulties in preparation for designing the new system.

Process

Following the initial requirements gathering meeting and a heuristic review of the existing Portal, the User Experience Team will meet with the core idea/development team on a weekly basis throughout the month of May for process clarification, prototype review, and for declaring next steps. In addition to weekly team meetings, the User Experience Team will meet with stakeholders to empathize with their use of the current system, and incorporate their experiences into the new design.

Due to my non-disclosure agreement, I will not be sharing actual images of the site as it was. Rather, all opinions and words are my own, and all prototypes are my own based on the original.

1| Heuristic Evaluation of Existing Site

Homepage

At first glance, there seems to be a lot of information on the page. There are a lot of bold, bright colors, which increase the difficulty of finding the correct task. There is navigation both on the top and left-side of the page, so I'm not sure where I should be looking to complete a task. If I wanted to submit an idea, I am not sure where I would actually do that.

The news page seems like it could be an important way to highlight information and resources that would encourage someone to submit an idea or encourage someone to learn more about ideas. I also like being able to see the active campaigns - this should be a larger focus on the page.

My Ideas Page

This page is meant to show the user all of the ideas that they have submitted. It seems like a good opportunity to add a "Submit an Idea" button.

Recommendations

This page is meant to show the user ideas that have been recommended to them. When I come to the page, my first question is why are these ideas recommended to me? It would also be nice to see a brief description of the ideas before being required to click into them.

Innovation News

The News page seems like it would be beneficial if users were spending a lot of time looking for Innovation News when they come to the site. There is a lot of content here that could be better organized on another site.

Reviewers List

This link is part of the Idea Review process that all idea submissions pass through. The load time was fairly slow, and there is not enough feedback for the user to know they should wait for a file to download after clicking the link.

Businesses

This page is intended to teach idea submitters about the Lines of Businesses that they can submit their ideas to be a part of. This page seems essential, however the purpose is unclear upon first visit.

Metrics

This page has a lot of useful information, and is interestingly the only page I can go to that allows me to view submitted ideas. There is also no scrollbar on the page, so I am unable to see the information that is 'below the fold'. There is also a legal disclaimer on the top of the page that is easy to miss because it has the same visual treatment as multiple alerts on the page.

Contact Us

This tab opens up the same 'My Innovation Team' window that we have already seen before. If this information needs to be seen at all times, then we can centralize it within the application.

My Idea Portal

The collaborative features of this portal can be moved to the internal social networking site to improve collaboration, participation, and visibility. For example, calendar events can be created in the internal social networking site, which will allow people to actually register to the event.

It seems like the majority of the actual action is taken from this portal, which is only accessible through a small link on the main site. We can highlight the important actions from this new page in the portal.

There are a lot of options here for forms. We can make the form type simple enough to include the request type and description.

I understand the Businesses page to be incredibly important for the user to know where they should submit their ideas. However, these two pages are two separated for the user to remember this information, We should include some helping information on the new design.

Other screens

  • My Subscriptions: Users are able to subscribe to receive updates on idea submissions they are interested in.
  • Ideas Referred to Me: Users are able to view ideas that others have recommended to them.
  • Your Innovation Team: A page displaying the team that works to coordinate the idea submission process.
  • Employee Recognition: A page to show past award winners and idea success stories.
  • Organization Chart: Lists the company's reporting structure.
  • Ideas: View of all ideas that have been submitted.
  • Campaigns: Ongoing 'themed' idea submissions.
  • FAQ: Guidance on how to submit ideas for users.

2| Empathize with the User

User Interviews

To get an understanding of the current user experience, I organized user interviews with a number of frequent users of the system ("Innovation Champions"), asking the following questions:

  • What is your line of business and role?
  • How long have you worked at the company?
  • How long have you been an Innovation Champion?
  • How did you become an Innovation Champion?
  • What does being an Innovation Champion entail for you?
  • How often do you visit the current innovation portal? What are you typically looking to do when you enter the portal?
  • What is your current experience submitting ideas via the current Innovation Portal?
  • What is your understanding of the current idea submission process?
  • How satisfied are you with your ability to view and track the status of your idea after it's been submitted?
  • What are your thoughts on the current process for tracking completion/execution of an idea?
  • What do you think might motivate an idea that previously hadn't before?
  • What are some things you think work well about the current submission process?
  • What are some things you think could be improved about the current submission process?
  • What are some things you think work well about the current idea portal?
  • What are some things you think could be improved about the current idea portal?
  • Have you made use of campaigns before? What has your experience been like thus far?

Understanding User Feedback

Below is a compiled list of notes, quotes, and general findings from the user interviews:

Notes

  • How to encourage users to submit ideas beyond monetary incentives?
  • Some submitters may not realize how much time it takes to submit ideas
  • Some submitters have expressed discouragement after submitting ideas yet have never heard anything - they may not want to submit an idea again
  • Approvers get emails when ideas are submitted - it is a fairly seamless process to approve ideas
  • Users make extensive use of Campaigns, not so much JAMs through the system. Users are much more likely to meet biweekly to discuss innovation.
  • Review current usage of the portal - initially people were submitting lots of ideas, but as the portal grew, approvers couldn't support the influx of ideas, resulting in user frustration.
  • One LOB has designed their own tool - a simple spreadsheet for tracking ideas specific to their own business. They are less focused on the use of the portal for their own company, and more focused on day to day innovation to improve quality of work and making employees more effective workers.
    • Spreadsheet provides immediate status and clear ownership - ideas are prioritized based on impact and taken into account at employee's year end review
    • Employees are encouraged to think about what they could do to improve their lives on a daily basis, and submit those as ideas. Not all of them are globally impactful, but the fact that ideas are being submitter starts a snowball effect - more confidence in submitting ideas - encourage others
    • Redesigned system is separated by years, ideas submitted are focused on the day to day difference
    • Simple fields - highlighting current workload vs new workload
    • Flat approval process, ideas move at their own pace
  • Meetings are help biweekly to discuss innovation for a business
  • User suggestion: The current application is cumbersome, it would be nice to separate it out into smaller sections, perhaps by business. There is transparency in the current system, to an extent, but it is difficult to be able to clearly see what someone else on your team or LOB is already doing and what ideas are already out there.
  • User wanted summary of submissions but found it difficult to pull information of ideas that have been submitted -> couldn't easily find summary data.
  • What does 'approved' mean once an idea is 'approved for idea development'? "There is no point in having movement of idea if no one owns it?"

Wishes

  • Make primary/secondary business selection easier to point the submitter in the right direction
  • Improve UI
  • Integrate into other systems
  • Transparency into Innovation Centers

Common Findings

  • Recognition/Incentives for employees to submit ideas
  • JAMs are not typically used as suggested by the portal. Many LOBs schedule their own in-person brainstorming/whiteboarding sessions.
  • Confusion between Campaign/JAM
  • Integration with other systems
  • Help the user during the submission process
    • Make it clear what primary/secondary Line of Business is
      • If LOB is selected incorrectly, it may not be looked at by anyone
      • User Suggestion: Send ideas to a common bucket, not a specific LOB Owner
      • No option to reassign/change LOB after rejected due to submission to incorrect LOB
    • Make the process transparent - what stage the idea is in, what the next step is, etc.
  • Users often get frustrated after they submit ideas, as they do not know what happens to them once they are submitted
  • The multiple-level approval is confusing - preferred would be a flat approval process
  • Found system difficult to use for idea submissions
  • Lots of emails from the portal about ideas that require no action
  • Almost every idea received requires offline interaction = less transparency
  • Track ideas in Excel
  • People may not submit ideas because they think it may have to be a big idea
  • After the annual idea competition ends, many ideas are "orphaned" - no clear owner with clear responsibility
  • Many ideas in the past are related to campaigns

Defining User Personas

Through user interviews, we identified five primary personas:

1| Submitter

Primary Tasks

  • Ability to view submitted ideas in the Idea Review stage
  • Ability to comment on ideas
  • Ability to upvote/downvote ideas
  • Ability to submit ideas

Level of Confidence with Portal: Low

  • Will need FAQs, access to LOB information to understand where to submit their ideas, access to information to understand differentiation between types of idea submission forms

2| Business Reviewer

Primary Tasks

  • Ability to review submitted ideas
  • Ability to comment on, approve, and reject ideas

Secondary Tasks

  • All tasks of Submitter persona

Level of Confidence with Portal: Low

  • User may not know how to review an idea

3| General Admin

Primary Tasks

  • Ability to configure email messages

Secondary Tasks

  • All tasks of Submitter persona
  • All tasks of Business Reviewer persona

Level of Confidence with Portal: Medium

  • Access is delegated by Innovation Champion

4| Innovation Champion

Primary Tasks

  • Ability to configure email messages
  • Ability to manage user access

Secondary Tasks

  • All tasks of Submitter persona
  • All tasks of Business Reviewer persona
  • All tasks of General Admin persona

Level of Confidence with Portal: High

  • Access is delegated by Overall Admin

5| Overall Admin

Primary Tasks

  • Ability to maintain and configure site
  • Ability to configure email messages
  • Ability to manage user access

Secondary Tasks

  • All tasks of other users

Level of Confidence with Portal: Expert

  • Represents "Super User" Persona

3| Define the Problem

After defining the user personas, the problem statements can be defined as follows:

  1. Submitters need to be able to easily submit ideas, track the status of ideas, and view submitted ideas.

  2. Business Reviewers need to be able to easily review submitted ideas, while receiving meaningful notifications of required action.

  3. General Admins need to be able to easily configure email messages, as well as approve ideas.

  4. Innovation Champions need to be able to run campaigns, configure user access, and review ideas.

  5. Overall Admins need to be able to maintain and configure the site, as well as manage user access and onboard and delegate new users of more experienced personas.

4| Ideate Possible Solutions

After listening to feedback from users, I drew out screen ideas for what the redesigned application could look like.

Features

The application should be accessible from the Intranet Homepage. To interact from the homepage, a portlet is displayed that allows the user to view top ideas and upvote or downvote them directly from the homepage.

The Idea Portal homepage displays featured ideas and calls to action for the user to get involved in idea submission and competitions. The "Submit Idea" button is clear and apparent at the top of all relevant pages.

The idea submission page is now simple and clear, with the status of the submission clearly displayed at the top of the page.

Ideas are now clear to find, with separations by idea type and submitter. Filters allow the user to clearly sort through all ideas to find the ones they're interested in.

Campaigns now have their own navigation tab and clearly show what campaigns are ongoing, when they end, and how to be a part of one.

The metrics page displays meaningful metrics, including idea submission by status, over time, and by business.
